# Phase 2 Plan 01: Build The Ingestion Runtime Foundation

## Objective

- **What:** Add the new `materials_discovery.data_sources` runtime, the
  canonical raw-source schema implementation, the snapshot/manifest/QA
  foundation, and the additive config seam required for provider adapters.
- **Why:** The current ingest path is too narrow to support multi-source staging.
  Phase 2 needs a stable runtime layer before any concrete provider adapters can
  be implemented safely.
- **Output:** A reusable runtime skeleton plus tests that lock the contract for
  the rest of the phase.
