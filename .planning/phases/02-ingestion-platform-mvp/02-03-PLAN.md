# Phase 2 Plan 03: Implement Direct API And OPTIMADE Adapters

## Objective

- **What:** Add the API-backed provider adapters and the shared OPTIMADE path
  needed to complete the Phase 2 MVP provider set.
- **Why:** Phase 2 must prove the framework supports both direct source-specific
  adapters and an OPTIMADE-family adapter path while preserving compatibility
  with the existing CLI and config shape.
- **Output:** A shared OPTIMADE base plus `Materials Project`, `OQMD`, and
  `JARVIS` adapters with deterministic offline tests and compatibility checks.
