# Phase 2 Plan 02: Implement HYPOD-X And COD Staging

## Objective

- **What:** Implement the first concrete provider adapters on top of the new
  source runtime: `HYPOD-X` as the direct snapshot-backed regression anchor and
  `COD` as the CIF-conversion proof path.
- **Why:** Phase 2 needs real providers to prove the runtime is not just a
  framework skeleton. `HYPOD-X` preserves scientific continuity; `COD` proves
  the design can ingest structure-rich sources.
- **Output:** Deterministic staged-source artifacts, manifests, QA reports, and
  tests for the direct snapshot and CIF-conversion families.
