Create `materials-discovery/src/materials_discovery/llm/launch.py`.

Implement:
- `resolve_campaign_model_lane(spec: LlmCampaignSpec, config: SystemConfig) -> tuple[list[str], str, LlmModelLaneConfig | None]`
- `materialize_campaign_seed(spec: LlmCampaignSpec, config: SystemConfig, launch_dir: Path) -> Path | None`
- `resolve_campaign_launch(spec: LlmCampaignSpec, config: SystemConfig, *, campaign_spec_path: Path, launch_id: str, artifact_root: Path | None = None) -> tuple[SystemConfig, LlmCampaignResolvedLaunch]`

Concrete resolution rules:

Lane selection:
- collect requested lanes from `spec.actions` in priority order `high`, `medium`, `low`, then `action_id`
- dedupe while preserving order
- choose the first lane present in `config.llm_generate.model_lanes`
- if no requested lane is configured, fall back to the baseline backend tuple only when the requested list is empty or includes `general_purpose`
- otherwise raise `ValueError("campaign spec requested no configured model lane")`

Prompt-conditioning:
- collect every non-blank `instruction_delta` in deterministic action order
- dedupe while preserving order
- if any action sets `preferred_max_conditioning_examples`, use the maximum of those values and the baseline `max_conditioning_examples`
- keep `example_pack_path` from the launch baseline unless a later phase adds a broader override surface

Composition-window:
- if an action provides `target_bounds`, copy those exact bounds into the resolved config
- otherwise, for each species in `focus_species`, shrink the existing range by exactly 10 percent of its current width around the midpoint
- leave species not listed in `focus_species` unchanged

Seed/motif variation:
- if `spec.launch_baseline.seed_zomic_path` is present, use it directly
- else if `spec.launch_baseline.example_pack_path` is present, load the eval set and materialize the first deterministic same-system example selected by existing conditioning selection logic into `seed_from_evalset.zomic` under the campaign launch directory
- else return a clear resolution error

Resolution output:
- deep-copy the config with the new composition bounds and resolved lane backend tuple
- do not write a rewritten YAML file
- return both the copied config and a fully populated `LlmCampaignResolvedLaunch`

Export the new helpers from `materials-discovery/src/materials_discovery/llm/__init__.py`.

Create `materials-discovery/tests/test_llm_launch_core.py` covering the rules above.
