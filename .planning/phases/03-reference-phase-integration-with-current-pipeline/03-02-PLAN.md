# Phase 3 Plan 02: Wire The Source-Registry Bridge Into `mdisc ingest`

## Objective

- **What:** Make `source_registry_v1` a real ingest path inside the existing
  CLI and attach staged-source lineage to the standard ingest manifest.
- **Why:** Projection alone does not help the current pipeline until
  `mdisc ingest` can call it behind the stable operator command.
- **Output:** A source-backed ingest branch that stages or reuses canonical
  source snapshots, projects them to processed reference phases, writes the
  usual summary/manifest, and keeps the legacy branch intact.
