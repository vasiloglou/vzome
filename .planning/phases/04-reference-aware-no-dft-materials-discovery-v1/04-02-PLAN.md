# Phase 4 Plan 02: Make Calibration, Ranking, And Report Outputs Comparable

## Objective

- **What:** Add the output-side benchmark/provenance layer that makes Phase 4
  runs comparable across source adapters and backend modes.
- **Why:** The pipeline already computes meaningful calibration and decision
  metrics, but it does not yet surface enough run context for operators to
  compare lanes cleanly.
- **Output:** Additive benchmark/reference context in manifests, a benchmark-pack
  summary artifact or exact equivalent, and richer rank/report provenance.
