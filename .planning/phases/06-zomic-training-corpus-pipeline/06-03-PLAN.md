- `candidate_to_zomic(record: CandidateRecord) -> CorpusExample`
- `build_record_conversion_trace(record: CandidateRecord) -> dict[str, Any]`

Implement a concrete v1 serializer with these rules:
- derive orbit names with `zomic_bridge._infer_orbit_name`
- sort sites by `(orbit_name, label, qphi tuple)`
- emit a deterministic script with:
  - a comment preamble containing `candidate_id`, `system`, and `fidelity_tier`
  - one `branch { ... }` block per orbit
  - one `label <sanitized_label>` statement for every site
  - a deterministic placeholder walk/body generated from the site's qphi tuple using a helper that emits a bounded axis-walk over the vocabulary `{red, blue, yellow, green, orange, purple}` and lengths `{short, medium, long, size N}`
- when no bounded walk is found within the chosen step limit, mark the example fidelity as `"approximate"` or `"heuristic"` instead of silently claiming `"exact"`
- preserve original label text in `properties["source_label_map"]`

The converter output must build a `CorpusExample` with:
- `source_family="candidate_record"`
- `source_record_id=candidate_id`
- `composition` copied from the candidate
- `properties["template_family"]`
- `properties["prototype_key"]` and `properties["zomic_design"]` when present in provenance

Create `materials-discovery/tests/test_llm_record2zomic.py` with tests for all behaviors above using realistic CandidateRecord fixtures.
