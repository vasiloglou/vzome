**Notebook 1: source_contribution_analysis.ipynb** (per D-11)
- Purpose: Which source contributed the most high-priority candidates for a given system
- Cells:
  1. Configuration: system_name, report_path (default: constructed from workspace_root)
  2. Load report JSON, extract entries
  3. Group entries by source (from evidence.calibration_provenance or benchmark_context.source_keys)
  4. Bar chart: count of high/medium/watch priority candidates by source
  5. Table: top candidates per source with hifi_score and xrd_confidence
  6. Summary cell with text output

**Notebook 2: cross_run_drift_detection.ipynb** (per D-11)
- Purpose: How results change between runs with different reference packs or timestamps
- Cells:
  1. Configuration: pack_a_path, pack_b_path
  2. Load both benchmark packs using LaneSnapshot.from_benchmark_pack
  3. Use compare_benchmark_packs to get ComparisonResult
  4. Grouped bar chart: gate results comparison (pass/fail for each gate)
  5. Side-by-side box plots or bar charts for key metric distributions
  6. Delta table: metric_name, mean_a, mean_b, delta, interpretation

**Notebook 3: metric_distribution_deep_dive.ipynb** (per D-11)
- Purpose: Stability/OOD/XRD distributions for a system, and how they shift across configurations
- Cells:
  1. Configuration: report_path (or multiple report paths for overlay)
  2. Load report entries
  3. Histogram: hifi_score distribution
  4. Histogram: stability_probability distribution
  5. Histogram: ood_score distribution
  6. Scatter: xrd_confidence vs xrd_distinctiveness
  7. Summary statistics table

## Prerequisites
- Python environment setup

- When to assemble a new pack vs. reuse cached
- Config structure for reference_pack members
- Expected artifacts

## 3. Pipeline Execution
### 3.1 Generate Candidates
- `mdisc generate --config configs/al_cu_fe.yaml`
### 3.2 Screen
- `mdisc screen --config configs/al_cu_fe.yaml`
### 3.3 High-Fidelity Validation
- `mdisc hifi-validate --config configs/al_cu_fe.yaml`
### 3.4 Rank
- `mdisc hifi-rank --config configs/al_cu_fe.yaml`
### 3.5 Active Learning (optional)
- `mdisc active-learn --config configs/al_cu_fe.yaml`
### 3.6 Report
- `mdisc report --config configs/al_cu_fe.yaml`
- Expected output: data/reports/ JSON

## 4. Benchmarking
### 4.1 Run Benchmark Suite
- `bash scripts/run_reference_aware_benchmarks.sh`
- Explain what it runs and expected artifacts
### 4.2 Benchmark Artifacts
- benchmark_pack.json structure and location
- calibration JSONs

## 5. Data Lake Operations
### 5.1 Index the Lake
- `mdisc lake index`
- What it produces: _catalog.json per directory, data/lake_index.json
### 5.2 Check Lake Stats
- `mdisc lake stats`
- Reading the output: artifact counts, stale entries
### 5.3 Compare Benchmark Runs
- `mdisc lake compare pack_a pack_b`
- Reading the comparison: gate deltas, metric distributions
- Example: comparing Al-Cu-Fe vs Sc-Zn

## 6. Analytics Notebooks
- Location: notebooks/
- How to launch: `jupyter notebook notebooks/`
- Overview of each notebook purpose
- Configuring notebook parameters

## 7. Troubleshooting
### Source Adapter Timeouts
- Symptoms, diagnosis, resolution
### Re-running Single Stages
- How to re-run just screen, validate, or rank
### Inspecting Stale Catalog Entries
- `mdisc lake stats` shows stale count
- Re-index: `mdisc lake index`
### Common Config Errors
- Missing system_name, wrong backend mode, missing benchmark corpus
### Pipeline Failures
- How to read manifest output_hashes for debugging
- Checking data/manifests/ for stage-level manifests

## 8. Quick Reference
- Table of all mdisc commands with one-line descriptions
- Key file paths
- Config file locations
