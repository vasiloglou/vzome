# Roadmap: Materials Design Program

**Milestone:** `v1.81`
**Focus:** Extensive LLM Tutorial and Programmatic vZome Visualization MVP
**Numbering mode:** continue after `v1.8` (`Phase 41+`)

## Milestone Summary

This milestone keeps the checked Sc-Zn tutorial anchor, but it deepens the
walkthrough in two specific ways:

- the guided tutorial and notebook become the extensive entry point to the
  repo's shipped LLM workflow families instead of a lighter companion path
- the final visualization step moves from a manual desktop-vZome handoff to a
  programmatic repo-owned path

The milestone is intentionally tutorial-first. It does not attempt to replace
desktop vZome wholesale or to add new LLM workflow families. It makes the
existing shipped surface easier to understand and easier to demonstrate end to
end.

## Phase Details

### Phase 41: Programmatic Visualization Artifact and Library Surface

**Goal:** Operators can refresh the checked Sc-Zn visualization artifact and
render it programmatically without manually opening desktop vZome.
**Depends on:** Phase 40

**Primary requirements**

- `VIS-01`
- `VIS-02`

**Success criteria**

1. A documented repo command or helper can refresh the checked tutorial
   visualization artifact from the existing Zomic source without a manual
   desktop-export step.
2. A standalone library or thin wrapper can load that artifact and render the
   checked design programmatically through one stable usage path.
3. The visualization surface is packaged clearly enough to be reused by the
   tutorial and notebook without inventing a general-purpose visualization
   platform.

### Phase 42: Extensive Guided Tutorial Expansion

**Goal:** Readers can follow one deeper Markdown walkthrough that keeps the
deterministic Sc-Zn evidence chain visible while explaining where the shipped
LLM workflows branch from it.
**Depends on:** Phase 41

**Primary requirements**

- `DOC-06`
- `DOC-07`

**Success criteria**

1. The guided tutorial explains the deterministic spine and the shipped LLM
   workflow families in one coherent operator story instead of a brief catalog.
2. The tutorial uses the new programmatic visualization path and keeps the
   geometry artifact chain explicit: `.zomic -> raw export -> orbit library ->
   candidates`.
3. The tutorial says clearly what desktop vZome still helps with, what the new
   viewer path replaces, and what remains future work.

### Phase 43: Notebook Visualization and LLM Walkthrough Integration

**Goal:** Operators can use the notebook as the most detailed executable or
previewable version of the same workflow, including programmatic rendering and
richer LLM branch guidance.
**Depends on:** Phase 42

**Primary requirements**

- `OPS-25`
- `OPS-26`

**Success criteria**

1. The notebook can render or launch the checked design programmatically with a
   documented preview-vs-execute path.
2. The notebook covers the shipped LLM workflow families with richer command,
   artifact, and branching guidance than the Markdown tutorial alone.
3. The docs index and cross-links explain when to use the Markdown tutorial,
   the notebook, and the standalone visualization library.

## Scope Boundaries

- This milestone does **not** replace desktop vZome with full browser or
  service parity.
- This milestone does **not** add new LLM algorithms, training flows,
  checkpoint mechanics, or campaign automation.
- This milestone does **not** broaden chemistry scope beyond the checked Sc-Zn
  tutorial anchor.
- This milestone does **not** quietly keep a required manual desktop step in
  the documented happy path.
